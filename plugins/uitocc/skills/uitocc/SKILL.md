# uitocc — 画面・音声コンテキスト参照

ユーザーが画面で見ている情報や聞いている音声を検索・参照する。uitocc watch デーモンが記録した画面履歴・音声文字起こしをMCPツールで検索する。

## いつ使うか

- ユーザーが「さっき見てた」「画面に出てた」「ブラウザで開いてた」「あのページ」など画面コンテキストに言及したとき
- ユーザーが「さっき聞いてた」「動画で言ってた」「ミーティングで話してた」など音声コンテキストに言及したとき
- ユーザーの発言の背景を理解するために画面・音声情報が役立ちそうなとき
- ユーザーが何かを参照しているが、具体的なURLやファイルパスを示していないとき

## 手順

1. まず `recent_screens` ツールで直近の画面状態を確認する（デフォルト10分）
2. 特定のキーワードで探す場合は `search_screen_history` ツールを使う
3. 音声に関する質問の場合は `recent_audio` で直近の文字起こしを確認する
4. 特定のキーワードで音声を探す場合は `search_audio` ツールを使う
5. 取得したコンテキストを基に、ユーザーの意図を理解して回答する

## MCP ツール

- `search_screen_history(query, minutes?, limit?)` — 画面テキストをキーワード検索。app名、ウィンドウタイトル、表示テキストが対象
- `recent_screens(minutes?, limit?)` — 直近N分の画面状態一覧
- `recent_audio(minutes?, limit?)` — 直近N分の音声文字起こし一覧
- `search_audio(query, minutes?, limit?)` — 音声文字起こしをキーワード検索

## チャンネルイベント

- `audio_transcript` — 10秒ごとにリアルタイムで音声文字起こしが届く。ユーザーが聞いている内容をリアルタイムに把握できる

## 注意

- uitocc watch デーモンが起動していない場合、ツールは「No screen/audio history available」を返す。その場合はユーザーに `uitocc watch` の起動を提案する
- 画面データはユーザーが許可したウィンドウのみ記録されている
- 音声キャプチャにはBlackHoleとマルチ出力デバイスの設定が必要
